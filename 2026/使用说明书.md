# Office 文档批量转换 & 梳理工具使用说明书

**版本**: v5.16.0  
**更新日期**: 2026-02-10

---

## 1. 软件定位

本工具用于把 Office 文档批量处理为 PDF，并支持合并、梳理去重，以及面向 NotebookLM 的来源反查。

核心价值：

- 批量转换（Word/Excel/PPT -> PDF）
- PDF 合并（分类拆卷 / 全部合一）
- 文件梳理（去重、索引）
- NotebookLM 溯源定位（`合并PDF+页码` 或 `短ID`）
- 本机检索增强（Everything / Listary）

---

## 2. 界面结构（双 Tab）

- `运行中心`（高频）
- `配置中心`（低频默认参数）

### 2.1 运行中心

用于本次任务执行：

- 运行模式：仅转换 / 仅合并 / 转换+合并 / 归集
- 路径：源目录、目标目录
- 执行参数：引擎、沙箱、合并参数、日期过滤
- NotebookLM 快速定位：`刷新Map`、`定位`、`打开文件`、`打开目录`、`Everything 搜索`、`复制 Listary 查询`

### 2.2 配置中心

用于长期默认值：

- 日志目录
- 超时与限制参数
- 排除目录与关键词
- Tooltip 样式（延迟、颜色、字体）
- 保存到 `config.json`

---

## 3. 数据流程（重点核对）

## 3.1 主流程 A：转换 / 合并

1. 输入：`source_folder` + `target_folder` + 运行参数  
2. 转换阶段（可选）：Office -> PDF  
3. 合并阶段（可选）：生成 `Merged_xxx.pdf`  
4. 输出目录：`<target_folder>/_MERGED/`  
5. 同步输出映射文件：
- `Merged_xxx.map.json`
- `Merged_xxx.map.csv`

映射文件记录每个源文件在合并 PDF 中的页码区间、短ID、路径等元数据。

## 3.2 主流程 B：NotebookLM 溯源定位

1. 你在 NotebookLM 拿到引用（常见是“合并文件 + 页码”）
2. 在 GUI 定位区输入：
- 主路径：`合并文件 + 页码`
- 备路径：`短ID`
3. 点击 `定位`
4. 程序在 `_MERGED` 中读取 `*.map.json`，回查源文件记录
5. 成功后可：
- 打开源文件
- 打开所在目录
- 用 Everything 二次检索
- 复制 Listary 查询语句

## 3.3 按钮灰态规则（你当前最关心）

`Everything 搜索` 和 `复制 Listary 查询` 在以下条件前会是灰色：

- 还没定位成功（没有 `last_locate_record`）

它们会在“定位命中后”自动亮起。

## 4. Map 数据契约（定位依据）

每条记录核心字段：

- `merged_pdf_name`
- `source_filename`
- `source_abspath`
- `source_md5`
- `source_short_id`
- `start_page_1based`
- `end_page_1based`
- `page_count`

定位逻辑：

- 按页码定位：命中 `start_page_1based <= page <= end_page_1based`
- 按短ID定位：命中 `source_short_id`

## 5. Everything / Listary 集成说明

## 5.1 Everything 集成

用途：在定位成功后，用本机索引快速检索对应文件。

配置节点（`config.json`）：

```json
"everything": {
  "enabled": true,
  "es_path": "",
  "prefer_path_exact": true,
  "timeout_ms": 1500
}
```

字段解释：

- `enabled`：是否启用 Everything 功能
- `es_path`：`es.exe` 绝对路径，留空则自动探测
- `prefer_path_exact`：是否带目录做精确检索（推荐 `true`）
- `timeout_ms`：命令超时毫秒

`es.exe` 自动探测顺序：

1. `es_path`（手动指定）
2. 系统 PATH（`es.exe` / `es`）
3. 常见目录：
- `C:\Program Files\Everything\es.exe`
- `C:\Program Files (x86)\Everything\es.exe`

诊断命令（Windows）：

```bat
where es.exe
```

## 5.2 Listary 集成

当前为轻集成：

- 点击 `复制 Listary 查询` 只会复制查询语句到剪贴板
- 不会自动控制 Listary 窗口

查询语句包含：`id/md5/file/path` 组合信息，便于你在 Listary 中粘贴后检索。

## 6. `config.json` 关键字段对照

| 字段 | 作用 | 示例 |
| :--- | :--- | :--- |
| `source_folder` | 默认源目录 | `"C:\\Docs"` |
| `target_folder` | 默认目标目录 | `"D:\\PDFs"` |
| `run_mode` | 默认运行模式 | `"convert_then_merge"` |
| `merge_mode` | 合并模式 | `"category"` / `"all_in_one"` |
| `merge_source` | 仅合并时扫描来源 | `"source"` / `"target"` |
| `enable_merge_map` | 是否生成 map | `true` |
| `bookmark_with_short_id` | 书签加短ID | `true` |
| `everything.enabled` | 启用 Everything | `true` |
| `everything.es_path` | `es.exe` 路径 | `"C:\\Program Files\\Everything\\es.exe"` |
| `everything.prefer_path_exact` | 精确检索 | `true` |
| `everything.timeout_ms` | 检索超时(ms) | `1500` |
| `listary.enabled` | 启用 Listary 辅助 | `true` |
| `listary.copy_query_on_locate` | 定位后自动复制查询 | `true` |
| `privacy.mask_md5_in_logs` | 日志脱敏 | `true` |
| `ui.tooltip_delay_ms` | 悬停提示延迟(ms) | `300` |

## 7. 快速核对清单（建议按此顺序）

1. 跑一次“转换+合并”，确认 `_MERGED` 下有：
- `Merged_xxx.pdf`
- `Merged_xxx.map.json`
- `Merged_xxx.map.csv`
2. 在定位区点击 `刷新Map`，确认有“已加载 N 个映射文件”提示
3. 用 `合并文件 + 页码` 做一次定位，确认命中并可打开源文件
4. 定位成功后，确认 `Everything 搜索` 和 `复制 Listary 查询` 不再灰色
5. 若 Everything 失败，先检查 `es.exe`：
- `where es.exe`
- 或在 `everything.es_path` 写死绝对路径

## 8. 常见问题（更新）

**Q1: 为什么 Everything / Listary 按钮是灰色？**  
A: 需要先成功执行一次“定位”。未命中前按钮会保持灰色。

**Q2: 定位输入页码后提示无效？**  
A: 页码必须是正整数，并且是“合并后 PDF 的页码”，不是源文件页码。

**Q3: NotebookLM 没给页码怎么办？**  
A: 用短ID定位（前提是来源文本或你的备注里带短ID）；否则先用文件名/关键词在 Everything 或 Listary 辅助检索。

**Q4: 改了配置不生效？**  
A: 在配置中心点“保存配置”，然后重新运行任务；运行中心参数会覆盖本次执行。
