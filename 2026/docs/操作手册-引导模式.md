# 引导模式操作手册（详细版）

> 适用版本：2026 工程当前分支  
> 目标：按“我想得到什么结果”来配置，而不是记配置键名。

---

## 1. 先看结论

1. 先选运行模式，再选输出目标，再看高级参数。
2. 所有配置控件都应有 Tooltip（鼠标悬停可见作用说明）。
3. 灰色控件代表“当前模式不生效”，不是坏了。
4. 运行前若存在模式冲突，系统会自动校正并在日志里给出摘要。

---

## 2. 标准操作顺序（建议每次都按这个顺序）

1. 在 `运行模式` 里选择本次目标：
   - `Convert Only`
   - `Convert + Merge`
   - `Merge & Convert`
   - `Collect / Index`
   - `MSHelp API Docs`
2. 在 `Output Controls` 里选择输出类型：
   - `Output PDF`
   - `Output Markdown`
   - `Merged Output`
   - `Independent Output`
3. 如果你开启了 `Merged Output`，去 `Merge Settings` 选择：
   - `By Category`（按分类并按阈值分卷）
   - `All In One`（单文件，不按阈值分卷）
4. 只在本次模式相关的区域改参数，灰色区域不用看。
5. 点击 `Start`，先看日志开头的配置摘要，再看进度。

---

## 3. 常用场景的最短路径

## 3.1 我要“转换并合并，80MB 分卷”

1. 模式选 `Convert + Merge`。
2. `Output PDF=开`，`Merged Output=开`。
3. `Merge Logic` 选 `By Category`。
4. `Max MB` 填 `80`。
5. 开始运行，日志里应出现“按分类/按阈值分卷”语义。

## 3.2 我要“只生成一个总 PDF”

1. 模式选 `Convert + Merge` 或 `Merge & Convert`。
2. `Merged Output=开`。
3. `Merge Logic` 选 `All In One`。
4. 此时 `Max MB` 即使有值，也不参与分卷。

## 3.3 我要“只做增量转换，不合并”

1. 模式选 `Convert Only`。
2. `Independent Output=开`，`Merged Output=关`。
3. `Enable Incremental Scan=开`（按需再开 hash 校验）。

---

## 4. 每个分区改什么

## 4.1 Shared Settings

1. `Source Dir/Target Dir`：本次输入输出位置。
2. `corpus.json`：是否生成运行产物清单。

## 4.2 Convert Settings

1. 引擎：WPS / MS Office。
2. Sandbox：隔离临时目录及低空间策略。
3. 策略与日期过滤：控制本次扫描范围和处理策略。

## 4.3 Merge Settings

1. 合并模式：`By Category` vs `All In One`。
2. Merge Source：只在特定模式下可改。
3. Merge Index / Excel List：合并附加产物。

## 4.4 Output Files

1. LLM 上传目录策略。
2. 上传清单、去重策略。

## 4.5 Config Center

1. 这里是“默认配置”，不是仅本次临时改动。
2. 区域保存按钮只保存该分区。
3. 顶部保存按钮保存完整配置档案。

---

## 5. 灰色控件说明（重点）

1. 灰色 = 当前模式不会读取该配置。
2. 值不会丢失，模式切回后仍可用。
3. 常见例子：
   - `collect_only` 下，合并和转换参数应灰色。
   - `all_in_one` 下，`Max MB` 不参与分卷。

---

## 6. 运行前自检清单

1. 源目录至少 1 个，目标目录有效。
2. 输出开关和运行模式一致。
3. 合并需求明确：
   - 要分卷 -> `By Category + Max MB`
   - 要单包 -> `All In One`
4. 日志目录、超时和沙箱参数合理。

---

## 7. 常见困惑与判断

1. 看到 `Max MB=80` 就一定分卷吗？
   - 不一定，`All In One` 不分卷。
2. 为什么我改了配置但没生效？
   - 先确认该控件是否灰色；灰色项不参与当前模式。
3. 配置被系统“改了”？
   - 这是模式收敛的自动校正，日志会给出摘要。

---

## 8. 建议的个人使用习惯

1. 每次开始前先看 Tooltip，不靠记忆。
2. 把常用场景存成配置档案。
3. 长期任务尽量用任务模式，不用手工来回改参数。
