# 引导模式流程文件（执行与校正）

> 适用版本：v5.20.0
> 配套阅读：[操作手册-引导模式.md](操作手册-引导模式.md)
> 目标：把「用户选择 → UI 状态 → 运行配置 → 实际执行」的链路固定下来，减少误判。

---

## 1. 总览：一次任务从向导到执行

```mermaid
flowchart LR
    U[用户在向导里勾选] --> O[overrides 写入 task_<id>.json]
    O --> L[运行时载入 project_config + overrides]
    L --> CO[配置收敛<br/>按 run_mode 过滤 / 校正]
    CO --> ER[effective_runtime_config]
    CO --> IA[inactive_config<br/>当前模式不读]
    CO --> CM[coercion_messages<br/>自动校正记录]
    ER --> EXE[执行流水线]
    CM --> LOG[日志摘要]
    IA --> GREY[UI 灰显 tooltip]
```

- 向导阶段：用户只决定 **overrides**，不动项目默认。
- 运行阶段：系统合并 `project_config + overrides`，再按 `run_mode` 做 **收敛**，产出三份东西：真正生效的 `effective_runtime_config`、被判定不读的 `inactive_config`、自动校正的 `coercion_messages`。
- 用户可见的 UI 灰显与日志摘要都来自这三份产物。

## 2. 模式到可编辑区域映射

```mermaid
flowchart TD
    M{run_mode}
    M -->|convert_only| CO[可编辑: 转换/增量/过滤/输出格式/独立输出<br/>灰显: 合并策略/merge_source/collect/mshelp]
    M -->|convert_then_merge| CTM[可编辑: 转换 + 合并参数<br/>强制: merge_source=target]
    M -->|merge_only| MO[可编辑: 合并参数/输出格式/merge 子模式<br/>灰显: 转换引擎/沙箱/增量扫描/日期过滤]
    M -->|collect_only| COL[可编辑: collect 参数/扩展名<br/>灰显: 转换参数/常规合并参数]
    M -->|mshelp_only| MS[可编辑: MSHelp 参数<br/>灰显: 常规转换/合并参数]
```

- 灰显不等于"被删除"：字段的值仍保留，切回兼容模式会恢复可读。
- 「编辑 = 不读」，避免用户在 collect_only 下修改合并参数而误以为生效。

## 3. 合并区内部联动

```mermaid
flowchart TD
    MG{Merged Output}
    MG -->|关| OFF[合并策略全部灰显]
    MG -->|开| SUB{Merge Logic}
    SUB -->|All In One| AIO[Max MB 显示但不参与分卷<br/>灰显 + tooltip]
    SUB -->|By Category| CAT[Max MB 参与分卷<br/>必须 >= 1]
    CAT --> VAL{Max MB 合法?}
    VAL -->|是| OK[按阈值分卷]
    VAL -->|否| FALLBACK[回退默认值 + 校正消息]
```

## 4. 运行前配置收敛（系统视角）

```mermaid
flowchart LR
    IN1[UI / overrides] --> MERGE[合并项目默认]
    IN2[run_mode] --> MERGE
    MERGE --> COERCE[收敛器]
    COERCE --> EFF[effective_runtime_config]
    COERCE --> INA[inactive_config]
    COERCE --> MSG[coercion_messages]
```

### 4.1 必做校正规则

1. `run_mode == convert_then_merge` → 强制 `merge_source = target`。
2. `output_enable_merged == false` → 合并区所有字段进入 `inactive_config`。
3. `merge_mode == all_in_one` → `max_merge_size_mb` 进入 `inactive_config`。
4. `merge_mode == category_split` → 校验 `max_merge_size_mb >= 1`；非法则回退默认值并记录到 `coercion_messages`。
5. `run_mode == collect_only` → 复制模式强制启用 SHA256 内容去重（见 §5）。
6. `run_mode in (merge_only, mshelp_only)` → `global_md5_dedup` 进入 `inactive_config`（这两种模式不做内容级去重，只跳路径已存在的目标）。

## 5. 去重策略流程

```mermaid
flowchart TD
    R{run_mode}
    R -->|collect_only| SHA[按 SHA256 内容去重<br/>两级: 按大小分组 -> 组内哈希<br/>永远开启]
    R -->|convert_only / convert_then_merge| MD5{global_md5_dedup?}
    R -->|merge_only / mshelp_only| PATH[仅路径去重<br/>目标已存在则跳过]
    MD5 -->|开| MDON[按类型全局 MD5 去重]
    MD5 -->|关| MDOFF[不做内容级去重]
    SHA --> REP1[索引 Excel: Duplicates sheet]
    MDON --> REP2[日志 + 失败报告 JSON]
    MDOFF --> REP3[日志]
    PATH --> REP3
```

- 两级 SHA256：先按 `(size, ext)` 分组，再在组内算哈希 —— 避免对所有文件暴力哈希。
- MD5 按类型去重 = 同一 bucket（word/excel/ppt/pdf/cab）下相同 MD5 仅保留一份，跨类型不去重。

## 6. 日志与提示流程

```mermaid
flowchart LR
    START[启动任务] --> SUM1[日志: 配置摘要<br/>run_mode / 输出开关 / 合并策略]
    SUM1 --> HAS{有 coercion_messages?}
    HAS -->|有| SUM2[日志: 校正摘要<br/>字段 / 原因 / 生效值]
    HAS -->|无| RUN[开始运行]
    SUM2 --> RUN
    RUN --> PROG[运行中: 进度 / 统计]
    PROG --> END{成功?}
    END -->|是| OK[状态列: succeeded]
    END -->|否| FAIL[状态列: failed<br/>失败报告入口]
```

---

## 7. 异常处理流程

```mermaid
flowchart TD
    E{异常类型}
    E -->|输入校验失败<br/>目录/数字/日期| BLOCK[阻止启动<br/>高亮无效项]
    E -->|模式冲突<br/>字段彼此矛盾| FIX[自动校正 + 校正摘要<br/>不阻止启动]
    E -->|运行时资源不足<br/>沙箱空间等| POLICY{策略}
    POLICY -->|block| STOP[终止并报告]
    POLICY -->|confirm| ASK[弹窗确认是否继续]
    POLICY -->|warn| WARN[只警告不阻断]
    E -->|单文件转换失败| SKIP[记入失败报告<br/>继续下个文件]
    E -->|进程级致命错误| CP[写 checkpoint<br/>状态置 failed]
```

## 8. 回归测试最小集合

| 编号 | 场景 | 关键设置 | 预期 |
|------|------|----------|------|
| R1 | 单包不分卷 | `convert_then_merge` + `all_in_one` + `max_mb=80` | 不分卷；日志出现「max_mb 不生效」校正 |
| R2 | 分卷 | `convert_then_merge` + `category_split` + `max_mb=80` | 按 80MB 阈值分卷 |
| R3 | 仅合并 PDF→MD | `merge_only` + pdf_to_md | 走 PDF→MD 子流程，不触发 Office 转换链 |
| R4 | 复制 + 索引 | `collect_only` | 合并和转换参数灰显；SHA256 dedup 自动开 |
| R5 | 旧配置兼容 | 加载含冲突字段的旧 config | 校正摘要正确出现并成功执行 |
| R6 | 任务编辑复用向导 | 已有任务 → 编辑 | 预填正确，保存不新增 id |
| R7 | 定时运行 | 任务 + `HH:MM` | 到时触发，`tasks/schedules.json` 可见 |
| R8 | 断点续传 | 中途中断 + 重新运行 | 询问是否续传，基于 `task_<id>_checkpoint.json` 恢复 |
