# AGENTS 方案：第一性原理与自我审查（第二轮）

## 一、第一性原理：方案的真正目的（再确认）

**不可再约简的目标**：

1. **接手可知**：任何人/任何 AI 接手时，能在最短路径内知道「项目现状」和「接下来该做什么」。
2. **可追溯**：每次代码变更、每次测试结果都有记录，且与单一事实来源（约定文档）一致。
3. **可执行**：规则可被 AI 或人无歧义地执行；必要时可由工具（如 check_doc_sync）部分强制。

**由此推导出的必要手段**（缺一不可）：

- **唯一入口**：AGENTS.md 必须明确「去哪读现状」「去哪读待办」「改代码要更新哪几个文档」「跑测试要记到哪里」。若 AGENTS 不列「接手必读」的文档路径，则目标 1 无法达成。
- **必要更新**：代码变更 → 更新计划/交接；跑测试 → 更新测试汇总；导致 AGENTS 所列路径失效的变更 → 更新 AGENTS。
- **协作方式声明**（本项目）：人提需求、AI 开发；AI 做好的一切都要有记录；不能依赖人提醒更新 AGENTS。此为**增补**，不替代上述「必须记录」规则。

## 五、最终硬核清单（只保留可落地、无漏洞）

- **唯一入口**：AGENTS.md 必须包含（1）接手必读：列出交接文档、测试汇总、待办/下一步建议的路径；（2）代码路径约定与测试命令；（3）必要更新规则。

- **原则**：可追溯；文档为交付物；导致 AGENTS 所列路径或目录失效的变更须更新 AGENTS。
- **禁止项与执行口径**：保留；例外须在交接文档注明。
- **执行者/会话**：建议，不强制。**无自动化测试**：测试汇总可注明「无」或「待补充」。
