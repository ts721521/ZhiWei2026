# 将程序封装为 exe

## 方式一：PyInstaller（推荐）

### 1. 安装依赖

```bash
cd 2026
pip install pyinstaller ttkbootstrap
# 若需完整功能，再安装：
# pip install pywin32 pypdf openpyxl python-docx beautifulsoup4
```

### 2. 打包

**方法 A：用脚本一键打包**

```bash
cd 2026
python build_exe.py
```

脚本会自动把当前目录的 `config.json` 复制到 exe 同目录；完成后在 `dist/` 下会有带版本号的 exe，如 `OfficeBatchConverter_v5.17.0.exe`（版本号从 `office_converter.py` 的 `__version__` 读取，单文件）。

**重要：必须从 dist 运行**  
请运行 `dist\OfficeBatchConverter_v{版本号}.exe`（例如 `OfficeBatchConverter_v5.17.0.exe`），**不要**运行 `build\OfficeBatchConverter\` 下的 exe。`build` 是打包时的临时目录，运行会报错「Failed to load Python DLL / 找不到指定的模块」。

**方法 B：命令行手动打包**

同目录的 `office_converter.py`、`ui_translations.py` 等会随入口自动打入包，无需 `--add-data`。示例：

```bash
cd 2026
pyinstaller --name OfficeBatchConverter --onefile --windowed --noconfirm --clean ^
  --hidden-import ttkbootstrap --collect-all ttkbootstrap ^
  office_gui.py
```

### 3. 打包模式说明

| 选项 | 说明 |
|------|------|
| `--onedir` | 生成一个文件夹，exe + 依赖在一起，启动快、易排查（推荐） |
| `--onefile` | 单文件 exe，携带方便但首次启动较慢，且杀毒软件易误报 |
| `--windowed` | 不弹出黑色控制台，适合 GUI 程序 |

### 4. 分发

- 将 `dist/OfficeBatchConverter_v{版本号}.exe`（如 `OfficeBatchConverter_v5.17.0.exe`）发给用户即可。
- 若你希望预置配置，请把 `dist/config.json` 一起发给用户，并与 exe 放在同目录。
- 若未提供 `config.json`，程序首次运行会在 exe 同目录自动生成默认配置。

## 常见问题

1. **报错「Failed to load Python DLL」或「找不到指定的模块」**：说明运行的是 `build\` 下的 exe。请改为运行 `dist\OfficeBatchConverter_v{版本号}.exe`（如 `OfficeBatchConverter_v5.17.0.exe`）。
2. **缺少 DLL 或模块**：在 `build_exe.py` 里用 `--hidden-import 模块名` 补全。
3. **杀毒误报**：`--onefile` 更容易被误报，建议给 exe 做代码签名（需证书）。
4. **体积大/启动慢**：`--onefile` 首次启动通常慢于 `--onedir`，属正常现象。

5. **报错「No module named PyInstaller」**：说明你当前用于打包的 Python 解释器里没有安装 PyInstaller。
   - 必须用“同一个解释器”安装（你当前是 `Python313`）：

```powershell
C:\Users\admin\AppData\Local\Programs\Python\Python313\python.exe -m pip install -U pyinstaller
C:\Users\admin\AppData\Local\Programs\Python\Python313\python.exe -m PyInstaller --version
```

   - 然后再执行：

```powershell
C:\Users\admin\AppData\Local\Programs\Python\Python313\python.exe g:/TianShuai_DOC/GitHub/GPTVersion/2026/build_exe.py
```
