# 将程序封装为 exe

## 方式一：PyInstaller（推荐）

### 1. 安装依赖

```bash
cd 2026
pip install pyinstaller ttkbootstrap
# 若需完整功能，再安装：
# pip install pywin32 pypdf openpyxl python-docx beautifulsoup4
```

### 2. 打包

**方法 A：用脚本一键打包**

```bash
cd 2026
python build_exe.py
```

脚本会自动把当前目录的 `config.json` 复制到 exe 同目录；完成后在 `dist/OfficeBatchConverter/` 下会有 `OfficeBatchConverter.exe` 及 `_internal` 等依赖。

**重要：必须从 dist 运行**  
请运行 `dist\OfficeBatchConverter\OfficeBatchConverter.exe`，**不要**运行 `build\OfficeBatchConverter\` 下的 exe。`build` 是打包时的临时目录，运行会报错「Failed to load Python DLL / 找不到指定的模块」。

**方法 B：命令行手动打包**

同目录的 `office_converter.py`、`ui_translations.py` 等会随入口自动打入包，无需 `--add-data`。示例：

```bash
cd 2026
pyinstaller --name OfficeBatchConverter --onedir --windowed --noconfirm --clean ^
  --hidden-import ttkbootstrap --collect-all ttkbootstrap ^
  office_gui.py
```

### 3. 打包模式说明

| 选项 | 说明 |
|------|------|
| `--onedir` | 生成一个文件夹，exe + 依赖在一起，启动快、易排查（推荐） |
| `--onefile` | 单文件 exe，携带方便但首次启动较慢，且杀毒软件易误报 |
| `--windowed` | 不弹出黑色控制台，适合 GUI 程序 |

### 4. 分发

- 将整个 `dist/OfficeBatchConverter/` 文件夹压缩发给用户。
- 用户解压后运行 `OfficeBatchConverter.exe`。
- 若程序会读写 `config.json`，请把 `config.json` 放在 exe 同目录，或首次运行后会在当前目录生成。

## 常见问题

1. **报错「Failed to load Python DLL」或「找不到指定的模块」**：说明运行的是 `build\` 下的 exe。请改为运行 `dist\OfficeBatchConverter\OfficeBatchConverter.exe`。
2. **缺少 DLL 或模块**：在 `build_exe.py` 里用 `--hidden-import 模块名` 补全。
3. **杀毒误报**：使用 `--onedir` 并签名 exe（需证书）可减轻误报。
4. **体积大**：单目录约几十 MB 属正常，因包含 Python 运行时和 ttkbootstrap；用 `--onefile` 可成一个 exe 但体积相近且启动更慢。

5. **报错「No module named PyInstaller」**：说明你当前用于打包的 Python 解释器里没有安装 PyInstaller。
   - 必须用“同一个解释器”安装（你当前是 `Python313`）：

```powershell
C:\Users\admin\AppData\Local\Programs\Python\Python313\python.exe -m pip install -U pyinstaller
C:\Users\admin\AppData\Local\Programs\Python\Python313\python.exe -m PyInstaller --version
```

   - 然后再执行：

```powershell
C:\Users\admin\AppData\Local\Programs\Python\Python313\python.exe g:/TianShuai_DOC/GitHub/GPTVersion/2026/build_exe.py
```
